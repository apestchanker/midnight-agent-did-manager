# Licensing

## License Choice

This project is intended to be published as open source under the Apache License 2.0.

Apache-2.0 is a permissive open source license that allows commercial use, modification, distribution, and private use, while requiring preservation of license and attribution notices in redistributed copies and substantial portions of the project.

## Attribution Requirement

If you redistribute this project or derivative works, you must preserve:

- the original copyright notice
- the Apache-2.0 license text
- any applicable `NOTICE` content
- source attribution to this repository and its original authors

At minimum, redistributed versions must keep clear attribution in source form and in any accompanying documentation or notice file.

Recommended attribution format:

```text
This product includes software derived from Midnight Agent DID Manager.
Source: <repository-url>
```

## What This Means In Practice

If you:

- fork the repository
- publish a modified version
- embed substantial parts of the code in another project
- redistribute binaries or hosted builds derived from this project

you must not remove the upstream attribution and licensing notices.

## Repository Publishing Checklist

Before public release, the repository should include:

1. a root `LICENSE` file containing the full Apache License 2.0 text
2. a root `NOTICE` file if you want to make attribution language explicit for downstream redistribution
3. preserved copyright and authorship notices in relevant files where applicable

## Dependency Licenses

This repository depends on third-party packages under their own licenses.
Anyone redistributing this project is responsible for complying with those upstream dependency licenses as well.
